## Usage

`FloatingWindow` creates a draggable element with fixed position:

<Demo data={FloatingWindowDemos.usage} />

## Constrain to viewport

Use `constrainToViewport` prop to restrict element movement to the viewport boundaries.
If you do not set `constrainToViewport` prop, the element can be dragged outside the viewport:

<Demo data={FloatingWindowDemos.constrainToViewport} />

## Constrain offset

Use `constrainOffset` prop to set the offset from the viewport edges when constraining the element:

<Demo data={FloatingWindowDemos.constrainOffset} />

## Drag handle selector

`dragHandleSelector` prop allows specifying a selector of an element (or a group of elements) that should be used to drag floating window.
If not specified, the entire root element is used as a drag target.

`excludeDragHandleSelector` prop excludes elements within `dragHandleSelector` from the drag event.
In the following example, the close button is excluded from the drag event:

<Demo data={FloatingWindowDemos.dragHandleSelector} />

## Enabled prop

Use `enabled` option to enable or disable dragging:

<Demo data={FloatingWindowDemos.enabled} />

## Set position

Call `setPosition` function to set the position of the element programmatically.
This function accepts an object with `top`, `left`, `right` and `bottom` properties,
from which you should only specify two (for example, `top` and `left`, `bottom` and `right`).

<Demo data={FloatingWindowDemos.setPosition} />

## Lock axis

Use `axis` option to restrict movement to the specified axis:

<Demo data={FloatingWindowDemos.axis} />
