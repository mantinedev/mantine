## Values
Details at [Form values](/form/values/) and [Nested fields](/form/nested/).

### values
Getter for current form values.
```jsx
form.values
```

### setValues
Handler to set all form values.
```jsx
form.setValues(values)
```

### setFieldValue
Handler to set a field value at the specified path.
```jsx
form.setFieldValue('path', value)
```

### insertListItem
Inserts a list item at the specified path. An optional index may be provided to specify the position in a nested field. If an
index is passed where an item already exists, it will be replaced. If the index is larger than the current list, the element is
inserted at the last position.
```jsx
form.insertListItem('path', item);
form.insertListItem('path', item, 3);
```

### removeListItem
Removes the list item at the specified path and index.
```jsx
form.removeListItem('path', 0)
```

### reorderListItem
Swaps two items of the list at the specified path. If no element exists at the `from` or `to` index, the list doesn't change.
```jsx
form.reorderListItem('path', {from: 2, to: 0})
```

### reset
Resets `values` to initial state (`initialValues`), clears all validation errors, resets all form status flags.
```jsx
form.reset()
```

## Validation
Validation requires defined validation rules set by the `validation` prop within the useForm properties.
Details at [Form validation](/form/validation/)

### validate
Validates all fields and returns the validation results.
```jsx
form.validate()
```

### validateField
Validates the field at the specified path and returns the validation results.
```jsx
form.validateField('path')
```

### isValid
Similar to `validate` or `validateField` but doesn't set `form.errors` and just returns the `hasError` flag of the validation.
```jsx
form.isValid()
form.isValid('path')
```

## Validation errors
Validation errors occur when defined validation rules were violated, `inititalErrors` were specified in useForm properties
or validation errors were set manually.
Details at [Form errors](/form/errors/).

### errors
Getter for current form validation errors.
```jsx
form.errors
```

### setErrors
Handler to set validation errors.
```jsx
form.setErrors({ path: 'Error message' })
```

### setFieldError
Handler to set a validation error at the specified path.
```jsx
form.setFieldError('path', 'Error message')
```

### clearErrors
Clears all validation errors. This doesn't restore `initialErrors`.
```jsx
form.clearErrors()
```

### clearFieldError
Clears the validation error at the specified path.
```jsx
form.clearFieldError('path')
```

## Status
Details at [Touched & dirty status](/form/status/)

### isTouched
Returns a boolean that indicates if the user focused or modified any or the specified field.
```jsx
form.isTouched()
form.isTouched('path')
```

### setTouched
Overwrites the touched state of all fields.
```jsx
form.setTouched({ path1: true, path2: false })
```

### resetTouched
Clears all field touched states. This doesn't restore `initialTouched`.
```jsx
form.resetTouched()
```

### isDirty
Returns a boolean that indicates that any or the specified field value is not the same as specified in `initialValues`.
```jsx
form.isDirty()
form.isDirty('path')
```

### setDirty
Overwrites the dirty state of all fields.
```jsx
form.setDirty({ path1: true, path2: false })
```

### resetDirty
Clears all field dirty states. This doesn't restore `initialDirty`.
```jsx
form.resetTouched()
```

## getInputProps
Returns an object with value, onChange and error that should be spread on input.

As second parameter options can be passed.
- `type`: default `input`. Needs to be configured to `checkbox` if input requires `checked` to be set instead of `value`.
- `withError`: default `type === 'input'`. Specifies if the returned object contains an `error` property with
`form.errors[path]` value.
- `withFocus`: default `true`. Specifies if the returned object contains an `onFocus` handler. If disabled, the touched state
of the form can only be used if all values are set with `setFieldValue`.
```jsx
<TextInput {...form.getInputProps('path')} />
<Checkbox {...form.getInputProps('path', { type: 'checkbox' })} />
```
