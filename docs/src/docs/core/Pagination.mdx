## Controlled

To control component state provide `page` and `onChange` props:

```tsx
import { useState } from 'react';
import { Pagination } from '@mantine/core';

function Demo() {
  const [activePage, setPage] = useState(1);

  return <Pagination page={activePage} onChange={setPage} />;
}
```

## Accessibility

Pagination renders a regular button element. To include aria-label for screen reader support
provide `getItemAriaLabel` function to generate aria-labels:

```tsx
<Pagination
  total={20}
  getItemAriaLabel={(page) => {
    switch (page) {
      case 'dots':
        return 'dots element aria-label';
      case 'prev':
        return 'previous page button aria-label';
      case 'next':
        return 'next page button aria-label';
      case 'first':
        return 'first page button aria-label';
      case 'last':
        return 'last page button aria-label';
      default:
        return `${page} item aria-label`;
    }
  }}
/>
```

## use-pagination hook

If you need more flexibility @mantine/hooks exports [use-pagination](/hooks/use-pagination/) hook,
you can use it to create your own pagination component.

## TypeScript

### Component props type

Import component props type:

```tsx
import type { PaginationProps } from '@mantine/core';
```

`PaginationProps` extends `React.ComponentPropsWithoutRef<'div'>`, all other props are available under _Component props_ tab.

### PaginationStylesNames type

`PaginationStylesNames` type is a union with Pagination [Styles API](/theming/styles-api/) names:

```tsx
import type { PaginationStylesNames } from '@mantine/core';

const paginationClasses: Partial<Record<PaginationStylesNames, 'string'>> = {
  item: 'pagination-item',
  active: 'pagination-active',
};

<Pagination classNames={paginationClasses} />;
```

See full styles descriptions list under _Styles API_ tab.
