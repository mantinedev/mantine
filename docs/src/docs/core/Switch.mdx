## Usage

Switch is another variant of <GatsbyLink to="/core/checkbox/">Checkbox</GatsbyLink>
component. Use it to capture boolean value input from user.
For better accessibility set `label` prop, it will add associated label element.

<Demo data={SwitchDemos.configurator} />

## Sizes

Switch has 5 predefined sizes: xs, sm, md, lg, xl.
Size defines label font-size, input width and height.

<Demo data={SwitchDemos.sizes} />

You can get switch sizes by importing `SWITCH_SIZES`:

```tsx
import { SWITCH_SIZES } from '@mantine/core';
```

<DataTable
  head={['Prop value', 'Width', 'Height']}
  data={Object.keys(SWITCH_SIZES).map((size) => [
    size,
    `${SWITCH_SIZES[size].width}px`,
    `${SWITCH_SIZES[size].height}px`,
  ])}
/>

## Radius

Radius controls border-radius of body and handle.
xs, sm, md, lg, xl radius values are defined in <GatsbyLink to="/theming/extend-theme/#spacing-radius-and-shadows">theme.radius</GatsbyLink>.
Alternatively, you can use a number to set radius in px:

```tsx
<Switch radius="lg" /> // -> theme predefined large radius
<Switch radius={10} /> // -> { borderRadius: 10 }
```

<Demo data={SwitchDemos.radius} />

## Controlled

```tsx
import { useState } from 'react';
import { Switch } from '@mantine/core';

function Demo() {
  const [checked, setChecked] = useState(false);
  return <Switch checked={checked} onChange={(event) => setChecked(event.currentTarget.checked)} />;
}
```

## Get element ref

You can get input ref with `elementRef` prop to:

```tsx
import { useRef } from 'react';
import { Switch } from '@mantine/core';

function Demo() {
  const ref = useRef();
  return <Switch elementRef={ref} />;
}
```

## Server side rendering

Component uses <GatsbyLink to="/hooks/use-id/">use-id</GatsbyLink> hook to generate unique ids and aria- attributes,
provide static `id` prop to prevent props mismatch:

```tsx
<Switch /> // -> random id generated both on client and server, props mismatch warning
<Switch id="my-accordion" /> // -> id is static, no mismatches
```

## Accessibility

Switch is a regular input with checkbox type. Provide `aria-label` in case you use switch without label for screen reader support:

```tsx
<Switch label="My switch" />; // -> ok, input and label is connected
<Switch />; // not ok, input is not labeled
<Switch aria-label="My switch" />; // -> ok, label is not visible but will be announced by screen reader
```
