## Disclaimer

**!important:** In most cases, you should not use Input component in your application.
Input component is a base for other inputs and was not designed to be used directly.

```tsx
// Incorrect usage, input is not accessible
<Input.Wrapper label="Input label">
  <Input />
</Input.Wrapper>

// Use TextInput instead of Input everywhere you want to use Input,
// it is accessible by default and includes Input.Wrapper
<TextInput label="Input label" description="Input description" />
```

## Usage

`Input` component is used as base for some other inputs ([Select](/core/select/), [TextInput](/core/text-input/), [Textarea](/core/textarea/), etc.).
The purpose of `Input` is to provide shared styles and features to other inputs:

## Input.Wrapper component

`Input.Wrapper` component is used in all other inputs ([TextInput](/core/text-input/), [Autocomplete](/core/autocomplete/), [MultiSelect](/core/multi-select/), etc.)
under the hood, you **do not need to wrap your inputs with it, as it is already included in all of them**.
Use `Input.Wrapper` only when you want to create custom inputs.

<Demo data={InputDemos.wrapperConfigurator} />

### inputContainer

With `inputContainer` prop you can enhance inputs that use `Input.Wrapper` under the hood,
for example, you can add [Tooltip](/core/tooltip/) to [TextInput](/core/text-input/) when
the input is focused:

<Demo data={InputDemos.inputContainer} />

### Input.Label, Input.Error and Input.Description

`Input.Label`, `Input.Error` and `Input.Description` components can be used to create custom
form layouts if default `Input.Wrapper` layout does not fit your requirements.

## Default props on MantineProvider

You can add [default props](/theming/default-props/) on [MantineProvider](/theming/mantine-provider/)
to `Input` and `Input.Wrapper` components. These default props will be inherited by all inputs
that use `Input` and `Input.Wrapper` under the hood ([TextInput](/core/text-input/), [Autocomplete](/core/autocomplete/), [MultiSelect](/core/multi-select/), etc.):

<Demo data={InputDemos.providerDefault} />

## Styles on MantineProvider

Same as with default props you can use `Input` and `Input.Wrapper` [Styles API](/styles/styles-api/)
on [MantineProvider](/theming/mantine-provider/) to add styles to all Inputs:

## Get input ref

```tsx
import { useRef } from 'react';
import { Input } from '@mantine/core';

function Demo() {
  const ref = useRef<HTMLInputElement>();
  return <Input ref={ref} />;
}
```
