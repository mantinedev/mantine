## Usage

Use Text component to render text and links with theme styles. Control Text styles with props:

- **size** – font-size from `theme.fontSizes` – xs, sm, md, lg, xl
- **color** – color from `theme.colors` – red, green, blue, etc.
- **weight** – font-weight property – 500, 700, bold, semibold, etc.
- **transform** – text-transform property – uppercase, lowercase, capitalize
- **align** – text-align property
- **variant** – text or link predefined styles

## Dimmed

Text supports special `dimmed` value in `color` prop. It sets color to `theme.colors.dark[2]` in dark theme and to `theme.colors.gray[6]` in light:

<Demo data={TextDemos.dimmed} />

## Gradient variant

To use gradient as Text color:

- set `variant` to `gradient`
- set `gradient` to `{ from: 'color-from', to: 'color-to', deg: 135 }`, where
  - `color-from` and `color-to` are color from `theme.colors`
  - `deg` is linear gradient deg

<Demo data={TextDemos.gradient} />

## Line clamp

You can specify maximum amount of lines with `lineClamp` prop. This option uses [-webkit-line-clamp](https://developer.mozilla.org/en-US/docs/Web/CSS/-webkit-line-clamp)
CSS property ([caniuse](https://caniuse.com/css-line-clamp)). Note that you cannot set `padding-bottom` on text element:

<Demo data={TextDemos.linesConfigurator} />

Line clamp can also be used with any children (not only strings), for example with [TypographyStylesProvider](/core/typography-styles-provider/):

<Demo data={TextDemos.lineClamp} />

## Inherit styles

By default Text always applies font-size, font-family and line-height styles,
but in some cases this is not a desired behavior. To force Text to inherit parent
styles set `inherit` prop. For example, you can highlight part of [Title](/core/title/):

<Demo data={TextDemos.inherit} />

## Custom component

By default text is rendered in div element, you can change it by setting `component` prop:

```tsx
import { Link } from 'react-router-dom';

<Text variant="link" component={Link} to="/hello">
  React router link
</Text>

<Text variant="link" component="a" href="https://mantine.dev">
  Normal link
</Text>
```

## TypeScript

### Usage with custom component

You need to pass additional type to use custom component with TypeScript
in case you need full type coverage for events and `ref`:

```tsx
// With element
<Text<'a'>
  component="a"
  onClick={(event) => console.log(event)}
  ref={(node) => {
    window.myRef = node;
  }}
/>

// With component
<Text<typeof Link>
  component={Link}
  onClick={(event) => console.log(event)}
  ref={(node) => {
    window.myRef = node;
  }}
/>
```

### Component props type

Import component props type:

```tsx
import type { TextProps } from '@mantine/core';
```

`TextProps` is a generic type, it accepts either a string or component type as single type argument:

```tsx
// TextProps with html tag, will extend React.ComponentPropsWithoutRef<'a'>
type AnchorTextProps = TextProps<'a'>;

// TextProps with custom component, will extend React.ComponentPropsWithoutRef<typeof MyOtherComponent>
type CustomTextProps = TextProps<typeof MyOtherComponent>;
```

See [TypeScript guide](/pages/basics/#typescript) to learn how to utilize props type with custom component.
