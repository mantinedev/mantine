## UMD builds deprecation

All `@mantine/` packages will no longer default to umd builds via `browser` field in `package.json`.
This change will simplify tree shacking for Webpack and should not affect your Next.js, Gatsby, Vite or CRA applications.

## react-jss peer dependency removal

All `@mantine/` packages no longer require react-jss installation as peer dependency â€“ it will be installed automatically as @mantine/core dependency.
Instead of direct react-jss usage @mantine/core now exports [createStyles](https://mantine.dev/theming/react-jss/) function
and a [set of components](https://mantine.dev/theming/next/) to implement server side rendering.

### createStyles function

New `createStyles` function lets you create styles with Mantine theme:

```tsx
import { createStyles } from '@mantine/core';

const useStyles = createStyles((theme) => ({
  button: {
    backgroundColor: theme.colors.blue[6],
    color: theme.white,
    padding: [theme.spacing.md, theme.spacing.lg],
  },
}));

function Demo() {
  const classes = useStyles();
  return <button className={classes.button}>My button</button>;
}
```

Note that it will only work if your application is wrapped with MantineProvider.

### Builtin server side rendering

@mantine/core package now comes with components to simplify server side rendering setup with Next.js and any other environment.
Checkout [Next.js guide](https://mantine.dev/theming/next/) to learn more.

## New features

- [Button](https://mantine.dev/core/button/#white-variant) component now supports new `white` variant
- [Text](https://mantine.dev/core/text/#inherit-styles) now supports font styles inheritance from parent element with `inherit` prop
- [Paper](https://mantine.dev/core/paper/) component now supports `withBorder` prop to add 1px border
- [ActionIcon](https://mantine.dev/core/action-icon/) component now supports `loading` state (same as in [Button](https://mantine.dev/core/button/) component)
- [SegmentedControl](https://mantine.dev/core/segmented-control/) component now supports setting data as an array of strings `<SegmentedControl data={['React', 'Angular', 'Svelte', 'Vue']} />`
- [NumberInput](https://mantine.dev/core/number-input/#decimal-steps) component now supports decimal steps and precision
