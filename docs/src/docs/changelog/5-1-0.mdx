## RichTextEditor formats

[RichTextEditor](https://mantine.dev/others/rte/) component now supports `formats` prop to restrict formats
that user can use in the editor.
In the following example three formats are enabled: `bold`, `italic` and `underline`,
toolbar includes `italic` and `underline` controls, `bold` format can be added with `Ctrl + B` keyboard
shortcut, other formats are not available:

<Demo data={RichTextEditorDemos.formats} />

## Minimal Next.js template

You can now use [minimal Next.js template](https://github.com/mantinedev/mantine-minimal-next-template)
that includes only basic server side rendering setup. It is useful when you want to set up your own tooltip
(configuration for Jest, Storybook, ESLint, prettier and other tools is not included).

## Other changes

- [New theme functions](https://mantine.dev/theming/functions/): `theme.fn.primaryShade` and `theme.fn.primaryColor`
- [Text](https://mantine.dev/core/text/) component now supports `strikethrough` and `italic` props to add text-decoration and font-style styles.
- [Text](https://mantine.dev/core/text/) component now supports `span` prop as a shorthand for `component="span"`.
- [Carousel](https://mantine.dev/others/carousel/) component now support keyboard events (switching slides with right/left arrows)
- [Accordion.Control](https://mantine.dev/core/accordion/) component now has `data-active` attribute when `Accordion.Item` is expanded
- [RichTextEditor](https://mantine.dev/others/rte/) now uses `value` instead of `defaultValue` to manage state, note that component does not support react strict mode
- [Spotlight](https://mantine.dev/others/spotlight/) now supports `disabled` prop to prevent spotlight rendering when the prop is set to `true`
- [Select](https://mantine.dev/core/select/) and [MultiSelect](https://mantine.dev/core/multi-select/) components now support `readOnly` prop
- [use-toggle](https://mantine.dev/hooks/use-toggle/) hook can now be called without options, it will use boolean values by default
- [Spotlight](https://mantine.dev/others/spotlight/) now supports `searchInputProps` prop that spreads props to search input
- [Popover.Dropdown](https://mantine.dev/core/popover/) now has `data-position` attribute with current position
