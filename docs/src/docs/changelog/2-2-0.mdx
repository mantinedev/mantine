import { MultiSelectDemos, HooksDemos } from '@mantine/demos';

### New components and hooks

New [MultiSelect](/core/multi-select/) component allows you to create searchable select
with option to customize labels and options:

<Demo data={MultiSelectDemos.countries} demoProps={{ toggle: true, zIndex: 4 }} />

[use-idle](/hooks/use-idle/) hook allows to detect if user does nothing:

<Demo data={HooksDemos.useIdleDemo} demoProps={{ toggle: true }} />

[use-fullscreen](/hooks/use-fullscreen/)use-fullscreen hook allows to work with Fullscreen API:

<Demo data={HooksDemos.useFullscreenDemo} demoProps={{ toggle: true }} />

New [use-move](/hooks/use-move) hook handles move behavior over any element,
you can use it to build custom sliders, color pickers, etc.:

<Demo data={HooksDemos.useMoveUsage} demoProps={{ toggle: true }} />

### Other changes

- [Menu](/core/menu/) and [RadioGroup](/core/radio-group/) components now works differently with Styles API – all styles and classNames should be
  added to root Menu/RadioGroup component instead of MenuItem/Radio
- [Card](/core/card/) component no longer has context styles for images and
  dividers, use new CardSection component instead
- @mantine/tag-picker package is deprecated – use [MultiSelect](/core/multi-select/) instead
- You can now customize shadow and padding in [Modal](/core/modal/) component
- [Button](/core/button/) now supports compact option which reduces its height
  and horizontal padding
- [Image](/core/image/) now supports adding figcaption
- New [Mark](/core/mark/) component – mark tag with theme styles
- [Menu](/core/menu/) component now supports labels in menu body and behavior
  customization (focus trap and close on scroll)
- 4 new premade pop-\* transitions were added to [Transition](/core/transition/) component
- New [use-logger](/hooks/use-logger/) hook will log given values each time component renders
- [Select](/core/select/), [NativeSelect](/core/native-select/) and [Autocomplete](/core/autocomplete/) components now support data as an array of strings
- New [use-shallow-effect](/hooks/use-shallow-effect/) hook – a drop in replacement for useEffect with shallow comparison instead of referential
- [use-scroll-lock](/hooks/use-scroll-lock/) hook now support state managing
  inside hook
