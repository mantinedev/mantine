## Breaking changes

- All inputs no longer support `inputStyle` and `inputClassName` props, this feature was replaced by [Styles API](https://mantine.dev/theming/styles-api/)
- Burger component no longer uses colors from theme, use
  hex color value instead
- Select component was renamed to NativeSelect
- `ElementsGroup`, `Hr` and `CardsGrid` components are replaced and no longer exported from @mantine/core package
- All inputs variant is set by default to filled in dark theme

## New theming options

- @mantine/core package now exports [GlobalStyles and NormalizeCSS](https://mantine.dev/theming/global-styles/) components to include global styles
- New [Styles API](https://mantine.dev/theming/styles-api/) allows you to style any part of Mantine components with inline styles or classes

## New @mantine/dates package

@mantine/dates s a new package that includes date pickers and calendars, it includes components:

[DatePicker](/dates/date-picker/) allows you to capture dates without free
user input. Component supports the same props as any other input from @mantine/core library and can be
used in forms:

<Demo data={DatePickerDemos.usage} demoProps={{ toggle: true, zIndex: 5 }} />

[DateRangePicker](/dates/date-picker/) allows you to capture dates range
from user:

<Demo data={DateRangePickerDemos.usage} demoProps={{ toggle: true, zIndex: 4 }} />

[TimeInput](/dates/time-input/) component allows to capture time input from
user:

<Demo data={TimeInputDemos.usage} demoProps={{ toggle: true }} />

[Calendar](/dates/calendar/) component allows you to capture date input fom
user or display events:

<Demo data={CalendarDemos.usage} demoProps={{ toggle: true }} />

Capture dates range input with [RangeCalendar](/dates/calendar/) component:

<Demo data={RangeCalendarDemos.usage} demoProps={{ toggle: true }} />

[Month](/dates/month/) component displays given month, it is the most basic
component which is used in all other components. You can use it to create your own date pickers and calendars
if Mantine components do not fit your requirements:

<Demo data={MonthDemos.usage} demoProps={{ toggle: true }} />

## New @mantine/prism package

New [@mantine/prism](https://mantine.dev/others/prism/) package exports `Prism` component that allows you to highlight code with [prism-react-renderer](https://github.com/FormidableLabs/prism-react-renderer) and your theme colors just like in Mantine docs.

## New @mantine/hooks hooks

- [use-did-update](https://mantine.dev/hooks/use-did-update) – useEffect replacement when you do not want hook to be called on mount
- [use-isomorphic-effect](https://mantine.dev/hooks/use-isomorphic-effect) – switch between useEffect during server side rendering and useLayoutEffect after hydration
- [use-force-update](https://mantine.dev/hooks/use-force-update) – force component to rerender
- [use-toggle](https://mantine.dev/hooks/use-toggle) – switch state between two values
- [use-uncontrolled](https://mantine.dev/hooks/use-uncontrolled) – manage state for both controlled and uncontrolled components
- [use-window-scroll](https://mantine.dev/hooks/use-window-scroll) – get current scroll position and handler to scroll to given position

## @mantine/core changes

- [Tabs](https://mantine.dev/core/tabs/) component now supports unstyled and pills variants
- All inputs now support `size` prop
- [Drawer](https://mantine.dev/core/drawer/) component now support title and close button
- [Tooltip](https://mantine.dev/core/tooltip/) component now displays tooltip on focus by default
- [Menu](https://mantine.dev/core/menu/) component now closes on scroll to prevent incorrect focus handling
- [Menu](https://mantine.dev/core/menu/) component now supports hover trigger
- 3 new premade transitions were added to [Transition component](https://mantine.dev/core/menu/): scale-y, scale-x and scale
- @mantine/core now exports new [UnstyledButton](https://mantine.dev/core/button/#unstyled-button) component that can be used to create custom buttons
