## RTL support improvements

- Emotion options on [MantineProvider](https://mantine.dev/theming/rtl/) now supports dynamic values. This feature allows to create dynamic direction changes.
- `@mantine/ssr` package now supports RTL with Next.js and any other ssr environment
- All components were tested to work correctly with [stylis-plugin-rtl](https://github.com/styled-components/stylis-plugin-rtl)
- Mantine theme object now includes `dir` property which must be set to `rtl` to support all features

## New hooks

- [use-validated-state](https://mantine.dev/hooks/use-validated-state/) – useState with state validation and capturing last valid value
- [use-event-listener](https://mantine.dev/hooks/use-event-listener/) – add event listener to an element using ref

## Other changes

- [Grid.Col](https://mantine.dev/core/grid/) component no longer requires to set `span`, it is set to Grid `columns` by default
- [Dropzone](https://mantine.dev/others/dropzone/) component now supports `onReject` callback which is called when user tries to drop files that do not meet size or mime type requirements
- [Stepper](https://mantine.dev/core/stepper/) component now supports `allowStepSelect` prop on `Stepper.Step` component which can be used to prevent user from reaching next steps before completing previous
- [Accordion](https://mantine.dev/core/accordion/) component now supports changing headings level with `order` prop
- [Tabs](https://mantine.dev/core/tabs/) component now supports getting tab keys in `onTabChange` handler
- [Slider](https://mantine.dev/core/slider/) default label transition is now set to `0` to make component feel more responsive
- [Portal](https://mantine.dev/core/portal/) now supports setting `target` with selector instead of element
- [Modal](https://mantine.dev/core/modal/) and [Drawer](https://mantine.dev/core/drawer/) components now support `target` prop to specify [Portal](https://mantine.dev/core/portal/) target container
- [TransferList](https://mantine.dev/core/transfer-list/) component now supports `limit` prop that can be used to improve performance of large data sets
- [Modals manager](https://mantine.dev/others/modals/) now supports `...others` props in `modals.openContextModal` handler
- [Select](https://mantine.dev/core/select/) and [MultiSelect](https://mantine.dev/core/multi-select/) components now support new `selectOnBlur` prop
- [Transition](https://mantine.dev/core/transition/) now supports `exitDuration` prop to specify unmount transition duration
- [Timeline](https://mantine.dev/core/timeline/) now supports `reverseActive` prop to reverse active items direction
- [Menu](https://mantine.dev/core/menu/) now supports `auto` size
- [use-move](https://mantine.dev/hooks/use-move/) hook now supports `rtl` direction
- [RichTextEditor](https://mantine.dev/others/rte/) now supports readonly state and code format
- All inputs components that are based on [Input](https://mantine.dev/core/input/) now support `iconWidth` prop

## 3.5.0 – 3.6.0 bug fixes

- Fix incorrect default `shouldCreate` logic in creatable [Select](https://mantine.dev/core/select/)
- Fix incorrect border-radius in [TransferList](https://mantine.dev/core/transfer-list/) component
- Add color style to [UnstyledButton](https://mantine.dev/core/button/#unstyled-button) in dark theme
- Fix incorrect [Menu](https://mantine.dev/core/menu/) body background color in dark theme
- Fix incorrect focus management in [DatePicker](https://mantine.dev/dates/date-picker/) which resulted in closing dropdown when anything inside was clicked
- Fix incorrect position calculation for [SegmentedControl](https://mantine.dev/core/segmented-control/) when scale transform is applied to parent node
- Fix [Slider](https://mantine.dev/core/slider/) initial value not being clamped with min/max
- Fix incorrect value set in [RangeSlider](https://mantine.dev/core/slider/) when user interacted with slider after value was force updated
- Fix possible permanent placeholder after hydration in [Image](https://mantine.dev/core/image/) component
- Fix jumping items in dropdown when item is selected in [Select](https://mantine.dev/core/select/), [MultiSelect](https://mantine.dev/core/multi-select/) and [Autocomplete](https://mantine.dev/core/autocomplete/) components, this fix is also applied to dropdowns with transitions
- Fix [Slider and RangeSlider](https://mantine.dev/core/slider/) components thumbs and marks overflowing parent element when is the same as min/max
- Fix value not being clamped in [DatePicker](https://mantine.dev/dates/date-picker/) with allowed free input, now value is clamped on blur
